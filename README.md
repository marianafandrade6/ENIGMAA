# ENIGMAA
Escopo Definido: o objetivo principal é o desenvolvimento de uma ferramenta capaz de processar e reverter métodos de codificação e cifragem específicos. 

Estimativas: Considerando um ciclo de deselvolvimento ágil, as horas foram divididas para garantir a robustez do código. 

|**ATIVIDADE**|**HORAS ESTIMADAS**|**PERCENTUAL**|

|Arquitetura e Setup|4h|10%|
|Codificação|16h|40%|
|Testes|14h|35%|

|**TOTAL**|40h|100%|

**PLANO DE RISCOS**: 

-**"Ausência de um integrante"**: Um membro do grupo pode faltar em dia de desenvolvimento do projeto ou apresentação. Plano de mitigação seria todos saberem e entenderem do projeto, compartilhar os códigos e anotações e sempre estudarem e compartilhar suas ideias e partes do trabalho, para caso isso aconteça, os outros integrantes saberem o que fazer. 

-**"Erros no código"**: O código do projeto pode ter alguma coisa de errado e não dar certo. A mitigação seria sempe testar, conferir os resultados desses processos sempre e revisar ente os integrantes para caso de errado eles terem consciência do que fazer. 

-**"Falta de organização da equipe"**: O grupo pode se perder durante o processo, sem saber o que fazer e como lidar com aquele problema. A ideia de mitigação seria organizar os papéis entre todos os integrantes, usar KanBan para agilizar o processo e sempre acompanhar o progresso do projeto.
